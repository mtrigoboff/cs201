CS201 Assignment 3 Instructions

Write code that forks into two processes, a parent process, and a child process.

The parent process will take the arguments to main() and send the characters in them one at a time to the child process through a pipe (one call to write for each character).

Do not send characters from argv[0] through the pipe. Start with argv[1] and continue through the rest of the arguments.

The child process will read the characters sent by the parent process one at a time, and will print out the number of characters it received from the parent. The child process should not use the arguments to main() in any way whatsoever.

The child should return normally (in other words, do not have the parent kill the child). It may be of use to know that read() will return immediately with a zero once the other end of the pipe is closed by the parent.

If I call your code this way: 

a03 abcd efg hi

The child process should print out:

CS201 - Assignment 3 - I. Forgot
child: counted 9 characters

Important: printing should be done only by the child process. The parent process should not print anything.

Your output should be formatted exactly like this (small differences in whitespace are OK). Remember to replace “I. Forgot” with your own name.

Make sure to have the parent process reap the child process before the parent process exits.

You can run a03.exe in Cygwin or a03 in Linux to see how this is supposed to work. Your code does not need to print out everything that these samples print out, just the child's line with the total number of characters.

Compiling Your Code

You are required to use these flags when compiling your code:

Required gcc Compiler Flags
flag effect of specifying flag
-m32 compiles 32-bit code, disallows use of 64-bit integer registers
-std=gnu99 allows use of newer features of C
-Wall compiler produces all possible warnings
-g includes information needed by debugger in object code

Here is how you would use these flags:

gcc -m32 -std=gnu99 -Wall -g yourcode.c -o yourcode
