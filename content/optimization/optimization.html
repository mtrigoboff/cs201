CS201: Optimization

“Optimization” typically means getting your code to either:

  • be smaller
  • run faster

In this lesson, we will concentrate on getting code to run faster.

Making code run faster is difficult and time-consuming. You only want to do this if you need to. To determine whether you need to, you need to measure the execution time of your code. Once you do this, you may decide that your code is too slow.

If you decide you need to make your code faster, you need to determine which part of your code needs to be faster. It is prohibitively expensive and counterproductive to attempt to make all of your code faster. What you need to do is profile your code to determine where you should focus your efforts (these places are known as “hot spots”). Simplicity is good. Know what your requirements are and optimize to that. Don't overdo it.

To summarize, to make your code faster, you need to:

  • measure your code’s speed
  • find the hot spots
  • optimize those hot spots

We will discuss each of these three tasks in this lesson.
