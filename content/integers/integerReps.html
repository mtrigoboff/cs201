The choice is up to the hardware designer. Modern computers support both the unsigned and two’s complement representations. Common sizes for integers are 8, 16, 32, and 64 bits.

Two’s complement representation is “hardware friendly.” When you add a negative and a positive two’s complement number, the normal process of addition that you learned in grade school will yield the correct arithmetical result.

Note that the leftmost bit corresponds to the sign of a two’s complement integer: 0 means positive, 1 means negative. We call that leftmost bit the sign bit.

Signed Overflow Error

In our 3 bit example, what happens when you add the two’s complement values 010 (2) and 011 (3)? You get the value 101, which is -3. 2 + 3 does not equal -3. What’s happened is that the numbers have “wrapped around” and produced a wrong result.

This is called an “overflow error.” An overflow error only happens when the carry into the sign bit position is not equal to the carry out of that position (by “carry” we mean what that word means in normal pencil-and-paper addition). Digital arithmetic hardware uses this condition to detect integer overflow errors.

Generating the Two’s Complement Negative of a Number

Two generate the negative of a two’s complement number:

  • flip the bits of the original number
  • add 1 to the result of this

Why this works:

  • flipping the bits and adding that to the original number produces all 1’s
  • add one to that, and you get all 0’s
  • therefore the method above generates the value that will produce zero when added to the original number
    (x + -x == 0 for all values of x)
