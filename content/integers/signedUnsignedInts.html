CS201: Signed/Unsigned Integers in C

Integer Data Conversion

There are some “interesting” issues in C when integers are converted or cast from signed to unsigned and vice versa.

  • in C, the default is signed (i.e. two’s complement)
    • you need to use the unsigned keyword if you want an unsigned integer
  • casting between signed and unsigned does not cause data conversion
    • consider a short (i.e. 2 byte) integer declared unsigned
      • if its value is 0xFFFF, it is interpreted as 65,535
      • if you cast it as signed, the very same value of 0xFFFF will be interpreted as -1
  • casting between integer and floating point data types will cause data conversion

suconv.c demonstrates these behaviors.

Sign Extension

When a shorter integer is moved into a longer variable (e.g. short to int), it will be either sign-extended (if it is signed), or zero-extended (if it is unsigned).

signext.c demonstrates this.

Combining Signed And Unsigned Integers

Combining signed and unsigned integers in the same expression can lead to unexpected results.

sucomb.c demonstrates an example of this.
