CS201: Random Access Memory (RAM)

Random access memory is one level up the memory hierarchy from disk storage. It is both faster and more expensive than disk storage.

Early Forms of Random Access Memory

  • Williams-Kilburn tube
  • core memory



    • the inventors of core memory at IBM needed to figure out how to make billions of little magnetic donut shapes. They turned for advice to the experts in the field.

Types of Random Access Memory

  • static RAM (SRAM)
    • expensive
    • retains data while power is supplied, “forgets” data when it loses power
    • data does not need to be continually “refreshed”
  • dynamic RAM (DRAM)
    • cheaper than SRAM
    • retains data while power is supplied, “forgets” data when it loses power
    • data needs to continually be “refreshed”
  • read only memory (ROM)
    • retains data without power
    • data can’t be changed once it’s written into the memory (write once)

Sequence of Hardware Operations for a Load

How the CPU gets data from RAM:

  • CPU puts requested address on address lines of bus
  • CPU toggles control line to signal RAM unit that a data request is ready
  • RAM unit reads address from address lines of bus
  • RAM unit places requested data on data lines of bus
  • RAM unit toggles control line to signal CPU that data is ready
  • CPU reads requested data from data lines of bus

# of address lines == # of data lines == word size
