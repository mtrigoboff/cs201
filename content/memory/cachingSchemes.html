CS201: Caching Schemes

“Caching” allows a system to perform faster by biasing its memory usage towards the upper (i.e. faster) levels of the memory hierarchy.

Caching is based on the probability that a computer’s use of memory often exhibits locality – if a computer accesses a particular address, it is likely to access addresses near that address. Locality can be both spatial (addresses are near each other in the address space) and temporal (accesses that are near each other in the address space will also tend to occur at times that are close to each other).

As we will see, caching is typically implemented at two different levels in the memory hierarchy:

  • between RAM and the disk, where it is called “virtual memory”
  • between the CPU and RAM, where it is called “caching”

Virtual memory was invented first, which is why its terminology differes from the terminology used in caching. But the concepts are the same. As is often true in Computer Science the same pattern can appear at multiple levels of an area of inquiry.
