Disk Concepts

In addition to reading about this topic in Bryant & O’Hallaron, look at these slides.

Here are some concepts about disk storage that you should understand for this course:

  • seek time: how long it takes the head to move to the designated track
  • rotational delay: how long it takes the disk to spin until the designated data is under the head
    • also called “rotational latency”
  • data rate: data density (bits/inch) * media speed (inches/second) → bits/second
  • transfer time: amount of data / data rate

Moving Data Between RAM and Disk

  • direct memory access (DMA)
    • the disk controller transfers data between RAM and the disk without the CPU having to be involved, which allows the CPU to do other things in parallel with the disk transfer
    • this happens via a “port,” a special location in memory that the disk controller watches for the presence of disk commands
    • DMA goes back a long way: there were “channels” that did DMA in IBM mainframes in the 1960’s
