Bryant & O’Hallaron Chapter/section 2.1.4 describes how numeric data can be stored in different byte orders on different machines. Consider the int value 0x12345678. This value can be stored in 4 consecutive bytes as either:

small endian:
0x78, 0x56, 0x34, 0x12
big endian:
0x12, 0x34, 0x56, 0x78

A two byte short value 0x1234 can be stored as:

small endian:
0x34, 0x12
big endian:
0x12, 0x34

Machines with small endian order: Intel, AMD (and other Intel-compatible), DEC.

Machines with big endian order: Motorola, IBM mainframe, Sun.

As in many other areas of Computer Science, there was (and probably still is) a religious war about this difference. Here is an amusing description of that war.

When It Matters

  • transferring data between two machines of different endian-ness
    • e.g. small-endian machine writes data to a file, big-endian machine tries to read that file
  • using a debugger to look at disassembled code on a small endian machine:
    • a 4 byte immediate operand of the value 3 will appear as 0x03000000

Code Example

Run endian.c on a small-endian machine, and you will see the difference.
