CS201: Numeric Data Sizes

Word Size

Computers all have a “word size,” which is the size of the “typical item” the machine can handle. Current computers have a word size of 32 bits, but 64 bit machines are becoming more and more common. The first microcomputers had a word size of 8 bits. When personal computers first became popular in the early 1980’s, the most common word size was 16 bits (although the Motorola 68000, which powered the first Macintoshes, was a clever hybrid of 16 and 32 bit word sizes).

A microprocessor with a 32 bit word size will have registers of that size, and data buses that can handle 32 bits simultaneously.

The word size of a processor determines the maximum size of the memory that can be addressed by that processor. A 32 bit processor can address a maximum of 232 (4,294,967,296) bytes.

Modern Pentiums (Pentia?) are very complex processors whose word size depends on which aspect of the machine you’re talking about. There are registers in a Pentium with sizes of 32, 64, 80, and 128 bits.

C Numeric Data Types

C does not define particular sizes for its integer data types. The idea is that int should be the “natural” data size for an integer on the target machine (“natural” means something like: most efficient for the machine to use). char and short will be a smaller size, and long will be a larger size. On current 32 bit machines, the sizes are:

length (bits)
char
8
short
16
int
32
long

Safe Usage

Since C’s integer data types are not of a defined size, you should never hard-code the current sizes in your code. If you need to use the size of an int, for instance, always code it as sizeof(int). Never code it as “4.”
