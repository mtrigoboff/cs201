Course Description

This course will introduce you to what’s “beneath the surface” of higher-level programming languages like C and C++, namely the “object code.” You will learn things about object code that will make you a more capable programmer. There’s no substitute for knowing what’s really going on “under the hood,” and this course will teach you about that.

In this course, we will mainly be working with Unix. Unix skills are valuable and acquiring them should increase your employment opportunities. If a job interviewer asks you to demonstrate some Unix skills, you don’t want to be sitting in front of that workstation wondering, “What do I do now?”

Unix is available to you in the following ways:

  • Windows PC
    • the department’s Linux machine, syccuxfs01.pcc.edu. You will be required to use this machine for Assignment 4.
    • download Cygwin (a Windows-based Linux emulator) and install it on your PC. Cygwin is also available on the lab machines in the Sylvania campus Computer Resource Center.
    • download Linux and install it on a PC you own. You can either make Linux the only operating system on your machine, or create a dual-boot setup so that you can run either in Linux or in Windows on the same machine.
  • Macintosh
    • modern Macs are Unix machines. Just open a command line window and you’re talking to Unix.
    • you will, however, not be able to use Mac Unix for this course unless you have an Intel-based Mac. This course concentrates on the Intel IA-32 architecture.

You will be using the C programming language in this course. All of the programming assignments should be done in C within a Unix environment.

In addition to acquiring skill with Unix, you should also develop proficiency with one of the source code text editors that are common in Unix. Your choices are:

  • vi
    • the power you need for programming
  • emacs
    • much more power than you need, has a steeper learning curve than vi (emacs is actually the programming language Lisp masquerading as a text editor)

I use vi, but you are free to choose emacs if you want something much more powerful. If you’re at that job interview and they find out that you can't do text editing in Unix, they will not be impressed.

Learning Outcomes

In t his course, you will you learn about the following topics:

  • how to do programming using Unix and the C language
  • what’s “under the surface” of your C/C++ code
    • object code (assembly code, machine language)
    • the machine itself (which will be studied more extensively in CS201)
    • what the machine is actually doing when it executes your code
  • deeper knowledge of how your programs work
    • special features of the Unix operating system
    • performance optimization
    • understanding of the true nature of the underlying Intel IA-32 architecture
  • Intel’s IA-32 Architecture
    • the Intel Assembly Language instructions commonly seen in object code
    • the machinery underlying the Intel IA-32 architecture
      • for instance, registers like eax are merely a convenient fiction
  • the structure and function of “the stack” (also known as “the calling stack”)
    • detailed understanding of “function calling overhead”
    • understanding of the nature of “buffer overflow attacks”
  • exceptional control flow
    • functions that return twice, functions that never return, etc.
  • techniques for optimizing your code to make it run faster
  • opportunity to practice and improve your programming skills

Textbooks

The textbook for this course is Computer Systems: A Programmer’s Perspective, Second Edition, by Randal E. Bryant and David R. O’Hallaron. It is available at the PCC bookstore.

You can get a used copy of the first edition instead, and save yourself some money.

Make sure to go to the textbook site’s errata page (second edition, first edition). You will save yourself hours of futile effort attempting to understand incorrect (and therefore incomprehensible) material. The 1st edition, in particular has a large number of errata, a number of which were contributed by me (including this one, which got me my very own page on their site).

If you need a reference to the C programming language, there are many available online. The GNU C Tutorial, for instance is usable, but occasionally it expresses strong opinions that I don’t necessarily agree with (e.g. regarding the goto statement).

If you want a book, The C Programming Language, 2nd Edition by Brian W. Kernighan and Dennis Ritchie is a classic of Computer Science. Kernighan and Ritchie are the creators of C.

Course Work

The work for this course will consist of online quizzes and programming assignments. The quizzes are available in the “Assessments” section of the course home page. The assignments are available in the “Assignments” section of the course home page. The assignments will sometimes (but not always) consist of a Regular part and a Premium part.

How To Submit Programming Assignments

Personalize Your Code

Your code should always print out the following as its first line of output:

CS201 - assignment - your name

Make sure you change the places marked in bold text to show me which assignment you are submitting and who you are.

For example, if I were submitting assignment 3, my code would print out:

CS201 - A03 - Michael Trigoboff

I will deduct IQ points if you submit an assignment to me and your code prints out “assignment” or “your name.”     :-)

What To Upload

Each individual assignment will specify what you have to upload.

Do not get “creative” with the file names. Name your files exactly as specified (this includes whether letters in the name are upper-case or lower-case). I use scripts to compile and run your code, and when your file names are not what’s expected, my scripts don’t work. Annoying your instructor in this way will most likely not produce the benevolent and merciful attitude that you hope for...

Grading

Your grade for this course will be a function of your grades on the quizzes and programming assignments.

Grading For Quizzes

You are allowed to take each quiz twice (it will be exactly the same quiz each time you take it). Your grade for a quiz will be the average of your scores for the two attempts. This means that if you get a perfect score the first time you take a quiz, there is no reason to take it the second time.

Grading For Assignments

You will get a maximum of either 100 or 200 points for each assignment. To get a good grade, you will want to maximize the number of these points that you earn.

How To Get 100 For An Assignment

To get 100 for an assignment, you must do the Regular part of the assignment in a satisfactory manner, which means your work must:

  • be coded according to the Coding Standards
  • compile without warnings (using gcc’s -Wall flag)
  • run without errors when given correct input and arguments. You do not need to check for input or argument errors unless:
    • such checking is specified by an assignment
    • you feel like doing it for your own learning
  • produce output exactly as specified by the assignment (I run your code using scripts, and non-standard output causes problems in that environment)
  • be submitted on time

How To Get 200 For An Assignment

To get 200 for an assignment, you will need to do the Premium part instead of the Regular part. For assignments that do not have an a Premium part, just do the Regular part – there is no way to get over 100 for an assignment that has no Premium part.

Standards for successful completion of a Premium part are the same as the standards for success in the Regular part, listed above.

Final Grade

Your final grade will be a function of the quality of your work on the assignments and your scores on the quizzes. To get an A, you will need to do all of the Premium assignments and score well on the quizzes. To get a B, you will need to do all of the Regular assignments and score well on the quizzes.

Netiquette

There’s a lot to say about “net etiquette,” but short summary is sufficient: I expect us to treat each other and this course with consideration and respect.

I want this to be a safe learning environment. There are no dumb questions. What’s dumb is to not ask when you don’t understand something. By not asking, you’re just wasting your tuition money and time.

The path to the top of the Mountain of Knowledge necessarily passes through the Valley of Stupidity. If you’re not willing to feel dumb on the way, you’re not going to learn much Computer Science. But in this class, we’re not going to do anything to make it worse than it has to be. There’s going to be plenty of opportunity to feel dumb, even if we’re nice to each other.

Academic Honesty

Learning to work with computers is the mental equivalent of learning gymnastics. If you don’t actually do the work yourself, you will not learn to do this - and the first time you actually have to accomplish something, it’ll be obvious that you don’t know what you’re doing.

It’s fine if you want to work with other students on a project, but make sure that you do the work yourself and understand what you did. Cheating and plagiarism will be dealt with as mercilessly as the college’s Academic Integrity Policy allows.
