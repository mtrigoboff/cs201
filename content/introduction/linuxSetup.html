Configuration Files

To set Linux up, you can add these files from ~michael.trigoboff/linux to your home directory:

  • .bash_profile
    • startup file for bash
    • adds . to your Bash PATH variable, which allows you to run your code without typing ./ before the program name
    • adds ~michael.trigoboff/bin to your Bash PATH variable, which allows you to run Bash scripts from that directory
    • reconfigures the Bash prompt so that it doesn’t take up an entire line all by itself
    • sets your Linux localization so that you get smart quotes in gcc error messages and warnings, instead of garbage characters
  • .vimrc
    • startup file for vim
    • sets tab stops to 4 (it takes more than one setting to do this)
  • .gdbinit
    • startup file for the gdb debugger
    • sets tab stops to 4

Compilation Script File

You will normally compile your source code with a command like this:

gcc <compiler flags> -o mycode

To avoid typing all of this over and over and over again, you can use the Bash script file that I use in class, gc. It’s available in the ~michael.trigoboff/bin directory, which should be part of your Bash PATH.

You can then compile your code (in file mycode.c) by doing this:

gc mycode
