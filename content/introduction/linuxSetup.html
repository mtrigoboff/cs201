CS201: Linux Setup

Configuration Files

To set Linux up on your machine, you should add these files to your home directory:

  • .bash_profile
    • startup file for bash
    • adds . to your Bash PATH variable, which allows you to run your code without typing ./ before the program name
    • reconfigures the Bash prompt so that it doesn’t take up an entire line all by itself
  • .exrc
    • startup file for vi
    • sets tab stops to 4 (it takes more than one setting to do this)
    • configures vi to replace tabs with spaces. You may not want this setting for all of your future use of vi, but please leave it this way during this class. By having spaces instead of tabs, your code will appear properly indented when typed out (using cat) in Bash, etc.

Compilation Script File

You will normally compile your source code with a command like this:

gcc -m32 -std=gnu99 -Wall -g mycode.c -o mycode

To avoid typing all of this over and over and over again, you can use the bash script file that I use in class, gc (it’s provided in a .zip file, which is ridiculous, but Desire2Learn seems not to be able to handle files whose names do not have an extension).

You will need to:

  • extract this file and placed it where bash can find it
    (i.e. in a directory that’s included in your execution PATH)
  • set that file to be executable, by executing the following Linux command:
    chmod 755 gc

You can then compile your code (in file mycode.c) by doing this:

gc mycode
