CS201: C Support for Console Input/Output

Your program can communicate with its user through input/output functions provided by Unix:

  • you can write formatted output to the console window by using printf()
  • you can read and format input from the user by using scanf()

printf

Compile and run printfdemo.c. Make sure you understand the various features of printf() that the code uses.

scanf

Compile and run scanfdemo.c. Make sure you understand the various features of scanf() that the code uses.

One thing to note is that if scanf() fails to parse what it's looking for (like if it's looking for a number and you type “XYZ”), those bad characters will stay in the buffer unless you explicitly flush them (see eatLine() in scanfdemo.c).

scanf() is currently “deprecated,” which means you should not use it in production code. But this is a learning situation, and scanf() is fine for our purposes. Besides, the currently “approved” method uses sscanf() which is quite similar to scanf(), and also uses getline() which is not defined in Cygwin. Finally, scanf() was “the standard” for quite awhile, and you may well run into code that uses scanf() in your career – so it’s useful to understand the in’s and out’s of scanf().
