In ASCII, characters are represented as a single byte. This allows for 256 different characters, which is enough for a language like English (26 small letters, 26 capital letters, 10 numerals, punctuation, etc). But 256 values are not enough for a lanuage like Chinese, where each word is its own different character. There are over 40,000 different Chinese characters. And what is really needed is a way to represent characters for all languages, so that you can have a document that displays characters from many languages at the same time. The answer to this problem is Unicode, a way of encoding characters as more than one byte when necessary. In C, characters such as this are known as wide characters, and are collected into character sets. There are data types like wchar_t and libraries that allow you to write C code that uses wide-character-based text.

In this class we will be dealing only with single-byte ASCII characters. But it’s important to know that this larger, multi-lingual situation is part of modern computing environments.
