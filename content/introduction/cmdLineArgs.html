CS201: Command-Line Arguments

This lesson will teach you to use command-line arguments in your C program. Suppose a user types this to the Unix command line:

> programName abc def 123

You can write code that will retrieve the number of arguments that were typed and the value of each one. In this case, the number is 4 ("programName" counts as one of the arguments), and you get the value of each argument as a C string. So, for instance, the 4th argument is a C string with the value "123".

In order to access these command-line arguments, code main() like this:

    main (int argc, char **argv)
    {
        ...
    }

argc contains the number of arguments (including the program name). argv is a pointer to an array of pointers. Each pointer in that array points to a C string containing the corresponding argument. (argv can also be coded like this: char *argv[]. The two forms of argv are equivalent.)

In the example above, argc will be 4, argv[0] will be "programName", argv[1] will be "abc", argv[2] will be "def", and argv[3] will be "123". Note that 123 is passed as a string, not as a number.

printargs.c is a simple C program that prints its command-line arguments.

There are two functions, atoi() and atof() (defined in stdlib.h) which are useful for translating numerical values contained in strings into actual numerical data (the names are abbreviations of alphabetic-to-integer and alphabetic-to-floating):

  • int atoi(char *str)
    • translates the number represented in str into an int
  • double atof(char *str)
    • translates the number represented in str into a double

Both of these functions will return zero if the string argument is not a valid number.

useargs.c is a simple C program that uses atoi() and atof() to parse its command line arguments and do something with them.
