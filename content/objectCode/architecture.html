CS201: Computer Architecture

Computer Architecture is the conceptual design and fundamental operational structure of a computer system.

Some traits of computer architecture:

  • Generally these are things visible to the programmer (a list of instructions, data types, memory access methods, etc.)
  • Protocols on how the microprocessor communicates to the rest of the system, i.e., how to read memory
  • Exceptions, modes, special cases

There is a huge amount of design behind this, such as how the arithmetic units are designed, what memory hierarchy is used, can instructions be executed in parallel, etc. Typically this stuff is considered microarchitecture, because it is not directly dependent to the programmer. The programmer cares, of course, but it's not the level that the programmer can access.

Computer Architecture is a course into itself, in fact more than one course. This material is just a brief introduction.

Key Components of the Computer

  • CPU: the “central processing unit”, i.e. the computer itself
  • register: a named memory location on the CPU. It holds data that is easily and quickly accessed. For example, if you write some code to add up a bunch of numbers, that subtotal, as it accumulates, is stored in a register.
  • register file: a group of registers organized in a way to easily access all of them.
  • ALU: the “Arithmetic Logic Unit” is the block of circuitry that performs adds, subtracts, shifts, compares, and other arithmetic and logical functions. It is usually connected to the register file so that any two register values can be input into the ALU and the output of the ALU can feed into any register file.
  • datapath: where data runs through the system
  • control unit: the logic which controls the datapath, access to memory, and which registers to access. Simply speaking, the control unit is the dispatcher and checker for all instructions running on the CPU.
