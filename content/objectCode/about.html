CS201: About This Module

What You Will Learn

How to understand the object code produced by the C compiler on an Intel IA-32 machine. Details of the architecture that you will need in order to understand the object code. The use of a debugger to view the object code created by compiling sample code.

In all of the topics in this lesson, refer to both Bryant & O’Hallaron and to Intel’s IA-32 Architecture Software Developer’s Manuals to understand the instructions in the object code.

As part of this lesson, you will do Assignment 2. There are three files mentioned in Assignment 2, which you can access from this page:

  • documentation for gcc’s inline assembler
  • asm.c
  • cpuid.c

Intel’s IA-32 Architecture Software Developer’s Manuals:

These are for use as references, to be used as needed. There’s no intention to have you read them like you would read a textbook.
  • Volume 1: Basic Architecture
    • describes the architecture and programming environment of IA-32 Intel Architecture processors
  • Volume 2A: Instruction Set Reference, A-M
    • describes the format of the instruction and provides reference pages for instructions (from A to M). This volume also contains the table of contents for both Volumes 2A and 2B
  • Volume 2B: Instruction Set Reference, N-Z
    • provides reference pages for instructions (from N to Z). VMX instructions are treated in a separate chapter. This volume also contains the appendices and index support for Volumes 2A and 2B
